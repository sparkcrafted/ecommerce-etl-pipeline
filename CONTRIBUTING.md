# Contributing Guidelines

Thank you for considering contributing to this project!

## How to Contribute

- Fork the repository and create a new branch for your changes.
- Describe all changes in clear commits with descriptive messages.
- Run all code linting and ensure no sensitive data is committed.

## Issues

- Please search open issues to avoid duplicating existing tickets.
- Open new issues with a clear summary and steps to reproduce.

## Pull Requests

- Reference the related issue (if applicable).
- Include tests or a clear test description if relevant.
- One functional area per pull request is preferred.
